# Setup notes: pod + VibeVoice

## Docker image

VibeVoice and both HuggingFace models are baked into the image ‚Äî no manual
pip install or model download needed on the pod.

Base image: `runpod/pytorch:2.4.0-py3.11-cuda12.4.1-devel-ubuntu22.04`
(PyTorch 2.4, CUDA 12.4, supports sm_50‚Äìsm_90 / all pre-Blackwell GPUs)

We tried `runpod/pytorch:1.0.6-cu1281-torch271-ubuntu2204` (PyTorch 2.7,
CUDA 12.8) to get Blackwell (sm_120) support, but most RunPod machines
have drivers too old for CUDA 12.8 and the container fails to start:
`nvidia-container-cli: requirement error: unsatisfied condition: cuda>=12.8`.
CUDA 12.4 works on a much wider range of machines.

Build and push from the **repo root** (bump version tag each time ‚Äî RunPod
won't re-pull if the tag is unchanged):

```
docker build --platform=linux/amd64 -f vibe/Dockerfile -t vibe:latest .
docker tag vibe:latest dockersaura/vibe:<vN>
docker push dockersaura/vibe:<vN>
```

Current image: `dockersaura/vibe:v7`

After pushing, update the RunPod template via the PATCH curl in
`runpod.md`. Current template: `e6qma5uqam` ("vibe v7").

The template must include `containerRegistryAuthId` for DockerHub auth ‚Äî
without it, pulls fail with `IMAGE_AUTH_ERROR: toomanyrequests`.
Auth ID: `cmqi6vbaq003mq3m4cb5bs2bl` (already set in current template).

## Starting a pod

```
cargo run -- new-pod gpu
```

Auto-selects cheapest available GPU with **>=24GB VRAM** and retries down
the price list on "not available" errors. Prints GPU, price, and pod ID.

**Wait ~60 seconds** after `new-pod` before running `listen-test` ‚Äî the
pod needs time to boot and get a public IP. Running too soon produces
`Error: pod <id> has no publicIp`.

No network volume ‚Äî attaching one locks the pod to a specific datacenter
region, limiting GPU availability. Generated audio is downloaded locally
after each run. **Do not restart a stopped pod** ‚Äî terminate and create a
new one instead.

SSH connects via direct IP+port (not the `<pod_id>@ssh.runpod.io` proxy):

```
cargo run -- ssh <pod_id>   # prints the ssh command
```

Terminate when done to stop billing:

```
cargo run -- terminate-pod <pod_id>
```

After `listen-test` completes, `watch-pod` is automatically spawned as a
background process. It sends a macOS notification every 10 minutes while
the pod is still running, as a reminder to terminate.

## Running a listen test

```
cargo run -- listen-test <segment_name> <pod_id> --gpu-price <price> --seed <seed>
```

- Normalizes `vibe/data/<segment_name>.txt` and uploads to the pod
- Runs VibeVoice inference (cfg_scale=1.3, speaker=Sarah)
- Downloads the generated `.wav` to `vibe/data/<segment_name>_generated.wav`
- Appends a row to `vibe/data/runs.jsonl` (GPU, price, duration, seed, etc.)
- Auto-spawns `watch-pod` on completion

Pass `--seed <N>` to fix the voice across multiple segments. Preferred seed
is **71463** ‚Äî see [voices.md](voices.md) for seed evaluations. Omit
`--seed` to let VibeVoice pick randomly (seed is captured and recorded).

## Segment files

`vibe/data/authorship_seg01‚Äì26.txt` ‚Äî the full authorship paper split at
paragraph boundaries with `Speaker 1: ` prefix on each paragraph.

- seg01‚Äì11: 250‚Äì400 words, generated by `split_authorship.py`
- seg12‚Äì16: 150‚Äì250 words, generated by `split_authorship_short.py`
  (resplit after seg07 clipping at ~400 words / 144s audio)
- seg16‚Äì26: 150‚Äì250 words, generated by `split_authorship_end.py`;
  seg26 is the References section

Short headings/fragments are merged into the following paragraph so they
don't produce TTS artifacts.

## Running multiple segments

`listen-test` handles one segment at a time. To run several in sequence:

```bash
for seg in seg12 seg13 seg14 seg15 seg16; do
  cargo run -- listen-test authorship_$seg <pod_id> \
    --seed 71463 --gpu-price <price>
done
```

## Stitching segments

Use ffmpeg with a file list (the `concat:` protocol only reads the first
file for WAV):

```bash
cd vibe/data
printf "file '%s'\n" seg12_generated.wav seg13_generated.wav ... > concat_list.txt
ffmpeg -y -f concat -safe 0 -i concat_list.txt -acodec copy stitched.wav
```

## Seed discovery workflow

Run a few segments without `--seed` to collect random seeds, then pick a
voice by listening:

```bash
for i in 07 08 09 10 11; do
  cargo run -- listen-test authorship_seg${i} <pod_id> --gpu-price <price>
done
```

Check `data/runs.jsonl` for the seed used in each segment. Preferred seed
for Sarah's voice is **71463** ‚Äî see [voices.md](voices.md).

## GPU requirements

**Minimum 24GB VRAM.** The 16GB RTX A4000 produces:
- Slower inference (RTF 0.45‚Äì0.87x vs 0.29‚Äì0.40x on 24GB cards)
- Hallucinations on repeated/similar phrases

`new-pod` enforces this automatically. See [quirks.md](quirks.md) for
full GPU performance data.
